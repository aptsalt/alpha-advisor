00What it is, in one breath

An advisor asks for a review. The system plans the work, retrieves from policy documents and a portfolio knowledge graph, calls a market-data tool, runs compliance checks, drafts a cited briefing, then pauses for advisor approval before finalizing — logging every step to a tamper-evident audit trail.

Agentic RAG

Retrieval that grades its own results and rewrites + retries on weak evidence — not one-shot RAG.

Vectors + graph

Prose via a vector store; portfolio structure via a client→holding→issuer→sector knowledge graph.

Governance both ends

PII redaction + policy block in; citation / PII / disclaimer checks out; 3 compliance checks with reasons.

Real HITL

LangGraph interrupt() + Command(resume=…). The agent proposes; the advisor disposes.

Tamper-evident audit

Append-only, hash-chained JSONL with a verify() that detects any edit or deletion.

Provider-portable

mock / ollama / azure by config — the graph is identical, so it can run fully on-prem.

01The orchestration graph

This is src/alpha/graph.py — a LangGraph StateGraph. Nodes are functions; edges are control flow. Two edges are conditional (diamonds): the agentic-retrieval retry loop, and the human-decision branch.

Colors: gold = governance, blue = RAG, teal = the agentic loop & the human gate, coral = a stop (refuse / discard).

g.add_edge(START, "plan")
g.add_edge("plan", "input_guard")
# a policy-blocked request never reaches retrieval / client data
g.add_conditional_edges("input_guard", route_after_input_guard,
                        {"blocked":"refuse", "ok":"retrieve"})
# agentic retrieval loop: weak evidence → rewrite → retrieve again; else proceed
g.add_conditional_edges("retrieve", should_retry,
                        {"rewrite":"rewrite", "proceed":"market"})
g.add_edge("rewrite", "retrieve")
# ... market → compliance → draft → output_guard → approval ...
g.add_conditional_edges("approval", route_after_approval,
                        {"finalize":"finalize", "discard":"discard"})

Why one typed state object

Every node receives the same AdvisorState (a TypedDict) and returns a partial update that LangGraph merges. The trace field uses an add reducer so each node appends a step. Because all state is explicit and typed, the run is inspectable and the audit trail is a byproduct, not extra work — in a regulated setting, explainability is the architecture.

class AdvisorState(TypedDict, total=False):
    request:str; client_id:str; intent:str
    safe_request:str; input_guard:dict          # governance
    query:str; retries:int; docs:list; retrieval_grade:float   # agentic RAG
    market:dict; compliance:list; compliance_status:str         # tools + checks
    draft:str; citations:list; output_guard:dict
    decision:str; advisor_note:str; final:str                   # human-in-the-loop
    trace: Annotated[list, add]   # reducer: each node appends its step

03Agentic RAG & Graph RAG

The single most-probed topic for this role. Know these three cold.

Naive RAG

embed query → top-k → stuff into prompt → generate. One shot, no self-check. Will answer confidently from the wrong context.

Agentic RAG

An agent decides what to retrieve, grades the results, and rewrites + retries on weak evidence. The difference between "plausible" and "defensible".

Graph RAG

Retrieve over a knowledge graph; answers multi-hop / structural questions (concentration, relationships, lineage) similarity can't.

The agentic loop — nodes/retrieve.py

def retrieve_node(state):
    # 1. pick sources — vector (prose) + graph (this client's structure)
    candidates = corpus.vector_store().search(query, TOP_K)
    if client_id: candidates += corpus.knowledge_graph().as_facts(client_id)
    # 2. grade each; keep only those above threshold
    graded = [{**c,"score":rel} for c in candidates
              if (rel:=_grade(query,c)) >= GRADE_THRESHOLD]
    return {"docs":graded, "retrieval_grade":mean, "retries":retries+1}

def should_retry(state):   # the conditional edge
    weak = state["retrieval_grade"] < GRADE_THRESHOLD
    if weak and has_budget and not state["docs"]: return "rewrite"
    return "proceed"

The graph — rag/graphrag.py

Schema: (Client)-[:HOLDS{weight}]→(Security)-[:ISSUED_BY]→(Issuer), and (Security)-[:IN_SECTOR]→(Sector), plus (Security)-[:RESTRICTED_BY]→(Policy). Sector hangs off the security, not the issuer — because one fund family issues both an equity and a bond fund. Getting that edge direction right is the kind of structural correctness a graph buys you.

def issuer_concentration(self, client_id):
    # aggregate portfolio weight per issuer — the multi-hop query vectors can't do
    agg = {}
    for h in self.holdings(client_id):
        agg[h["issuer"]] = agg.get(h["issuer"],0) + h["weight"]
    return agg   # {"Vantage Funds":0.78, "NorthForge Inc.":0.22}

Local backend is networkx (serverless). Production swaps to Neo4j via rag/neo4j_adapter.py — the same methods in Cypher, selected by ALPHA_GRAPH_DB=bolt://…, falling back to networkx if absent.

04Governance, guardrails & compliance

In regulated wealth management this isn't a feature you bolt on — it's what makes an autonomous agent deployable at all. Guards sit on both ends; compliance is three explicit checks with reasons.

input_guard (before the model)
  • PII redaction — the model never needs an SSN to reason about allocation; minimizing what it sees is itself a control.
  • Policy block — a prohibited request (e.g. "guarantee returns") routes straight to refuse and never touches retrieval or client data.
output_guard (before a human sees it)
  • Grounding — rejects a draft with no citations.
  • PII leak scrub — re-redacts anything that slipped through.
  • Disclaimer — appends the "not investment advice until approved" notice.

Three compliance checks — nodes/compliance.py

CheckQuestionOn Jane
suitabilityDoes the allocation match the documented risk profile? pass (64% equity vs 60% target, within tolerance)
restricted_listDoes any holding touch a restricted-list edge? FAIL — NorthForge Energy is restricted
concentrationAny single issuer over the policy limit? WARN — Vantage 78% > 25% limit

Worst finding sets compliance_status; findings become cited evidence in the briefing and are shown to the advisor at the gate. Autonomy with a hard human backstop — not instead of one.

05Human-in-the-loop — the interrupt

The advisor is always the last actor before the client. This is a real LangGraph interrupt(), not a prompt that says "ask the user".

from langgraph.types import interrupt

def approval_node(state):
    request = {"draft":state["draft"], "citations":state["citations"],
               "compliance":state["compliance"]}
    # execution PAUSES here; full state is persisted to the checkpointer
    response = interrupt(request)
    return {"decision":response["decision"], "advisor_note":response.get("note","")}

When the advisor decides, the caller resumes: app.invoke(Command(resume={"decision":"approved"}), thread). The value passed to resume becomes the return of interrupt(), and a conditional edge routes to finalize or discard.

Why this is the senior answer: the pause persists full graph state to a checkpointer. With a Postgres checkpointer that state survives a process restart and can be resumed by any replica — so a run can wait hours for an advisor and the service still scales horizontally. The interrupt isn't a UI trick; it's a durable workflow primitive.

06Auditability — tamper-evident by construction

src/alpha/audit.py. Every governed action appends one immutable JSONL line whose hash chains to the previous line, so a deletion or edit anywhere is detectable.

def log(event, payload, run_id=""):

Two complementary records: the audit log is the immutable compliance trail ("what was decided, on what evidence, by whom"); OpenTelemetry spans (telemetry.py, ALPHA_TRACING=1) are the operational trace ("what ran, how long, where it failed"). 09Interview Q&A — fast recall

Each answer is anchored to code you actually wrote. Full set in docs/interview-qa.md.

Naive vs Agentic RAG? Naive = embed→top-k→generate, one shot. Agentic = retrieval is a decision: grade results, rewrite & retry on weak evidence. In wealth management that's the line between a plausible and a defensible answer. (retrieve.py, should_retry edge.)
Vector vs graph — when each? Vectors for prose similarity ("what's the suitability policy?"); graph for multi-hop structure ("how concentrated is this client in one issuer?"). My demo catches a 78% issuer concentration that requires aggregating two funds — invisible to vector search. (graphrag.py.)
How does HITL work technically? LangGraph interrupt() persists full state to the checkpointer and surfaces an approval payload; I resume with Command(resume=…). With Postgres it survives restarts and any replica can resume — durable, scalable. (approval.py, api.py.)
How is it explainable / auditable? One typed state object; every node appends a trace step and writes a hash-chained audit record; every claim in the briefing is cited. For any output I can show what ran, on what evidence, what compliance found, and which human approved it. (audit.py.)
Deploy & scale on Azure? Stateless graph + durable checkpointer → containerize → Azure Container Apps, Azure OpenAI, Azure AI Search + Neo4j, OTel→Azure Monitor, audit→WORM storage. Horizontal because any worker resumes any paused run. (deployment.md.)
Preventing hallucinated numbers? Market figures come from a tool, not model text; the draft only cites provided evidence; the output guard rejects an uncited draft. PII never reaches the model. (market.py, guardrails.py.)
